In C++, computation is expressed in an expression which is composed of operators and operands.

The first expression that was shown so far was a stream insertion expression:

cout << "Hello world!\n";

Here, it's not apparent that computation is taking place; in fact, computation need not take place in an expression. What matters is that an operator is being used, this one being the stream insertion operator as denoted by <<.

Most expressions produce a result which is a value that can further be used by another operand within the same expression statement. The insertion operator returns it's left operand, thus one can "chain" additional insertion operations to the same object. The above statement could have been rewritten as:

cout << "Hello" << " " << "world!" << "\n";

Input/Output

An expression that uses the stream insertion operator does not "produce a result" in the sense of creating a new value, it returns the same value that was used for it's left operand whose type must be ostream to begin with.

The type of cout is ostream, it represents an output stream. A stream is an interface for transmitting data in an ordered sequence. An output stream only allows one to send data while an input stream only allows one to receive data. An Input/Output stream allows both. As you already know the stream insertion operator is used for sending data to an output stream.

cout, which stands for "character output", is associated with the program's standard output stream; it is one of the 3 standard streams that every program has as it's own.

For console applications it is usually the case that: the standard output stream (cout) is associated with the terminal window, the standard input stream (cin) is associated with the keyboard, and the standard error stream (cerr) is also associated with the terminal window.

cout and cerr are used the same way for different purposes, cout is for sending normal messages and cerr is for sending error messages. cin is for receiving data, usually from the user and the operator used for receiving data from an input stream is the stream extraction operator, symbolized by >>.

The following terms are roughly synonomous with "input": read, receive, extract, scan, get

The following terms are roughly synonomous with "output": write, send, insert, print, put

Computation

These are some of the arithmetic operators provided by C++, the column labeled "Syntax" shows how they can be used in an expression where a and b can be any subexpression of appropriate types.

Operator name Syntax Description
Unary plus +a Returns the positive value of a
Unary minus -a Returns the negative value of a
Addition a + b Returns the value of a plus b
Subtraction a - b Returns the value of a minus b
Multiplication a * b Returns the value of a multiplied by b
Division a / b Returns the quotient value of a divided by b
Modulo a % b Returns the remainder value of a divided by b

The arithmetic operators are usable on all arithmetic types which include int and double. Other types may support the use of certain arithmetic operators but the semantics of their use may entirely be different.

The unary plus and unary minus operators are known as unary operators because they each only require 1 operand. The other operators shown above are binary operators since they each require 2 operands.

The following unary operators are used for incrementing/decrementing the value of a variable; unlike the above operators, these operators modify the variable for which they are used on.

Operator name Syntax Description
Pre-increment ++a Increment a and return a
Post-increment a++ Increment a and return the old value of a
Pre-decrement --a Decrement a and return a
Post-decrement a-- Decrement a and return the old value of a

Operator precedence

The rules of operator precedence in C++ determine the order of the evaluation of subexpressions within an expression.

Generally: increment/decrement expressions are evaluated first, multiplication/division/modulo expressions are evaluated second, addition/subtraction expressions are evaluated third, and assignment is evaluated last.

The associativity of an operator determines the order of the evaluation of an operator's operands. The associativity of an operator is either Left-to-right or Right-to-left. For example, the expression a + b has Left-to-right associativity since a is evaluated first and then b. The associativity of a = b is Right-to-left since b is evaluated first and then a.
